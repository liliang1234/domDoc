【产品介绍】
  • 票之家支撑平台1.0版主要服务于票之家公司运营部门、财务部门的工作。
  • 运营部门在平台完成用户审核、权限分配、渠道分配工作。
  • 财务部门在平台完成财务基础信息的核查工作，后期可能扩展更详尽、智能的对账工作。
  • 另外将有权限模块实现成员在平台内的权限分配。

【名词定义】
  • 景区公司：景区门票、演艺票的供应商，一个景区公司下可能含有多个景区、演艺。
  • 旅行社：仅旅行社可开行程单，购买团票必须有行程单。
  • 旅行社部门：又称地接社部门，挂靠在旅行社使用旅行社的资质做业务，分强关系（直属旅行社）和弱关系（挂靠多家旅行社）两种关系。旅行社部门可能是公司性质，也可能是个人性质。
  • 导游：挂靠在旅行社或者旅行社部门，也分强关系和弱关系两种。
  • 客栈：住宿供应商，一个客栈下可能会有多个分店。
  • 商户：商铺等。
  • 运营商：帮票之家平台拓展用户的公司，可能拓展供应商、分销商，运营商必然是当前某一种注册角色的附加身份。
【模块说明-用户账号体系】
相关原型页面：注册、登录、重新申请页面。

用户注册
  • 注册角色有：旅行社、旅行社部门（公司）、旅行社部门（个人）导游、商户。
  • 注册入口：用户通用注册入口为piaozhijia.com，某一种的角色注册模块可能被调用到其它产品线使用。
  • 注册流程：


  • 旅行社注册。
  1. 手机号：仅13、14、15、18开头的11位半角数字允许获取验证码（报错信息：请输入正确的手机号），如果是已经注册的手机号提示「该手机号已注册，请直接登录」。点击「获取验证码」后该按钮禁用并进行120秒倒计时，120秒之后允许重新获取验证码。系统自动发送4位数的验证码，北京团队已有现成接口。
  2. 验证码：填写正确的验证码（报错信息：验证码错误）并「确认」之后，手机号、验证码两项锁死，并在页面下方呈现其它注册信息的填写界面。根据不同的注册身份呈现不同的注册信息填写界面，提交注册之前注册身份允许自由切换。系统验证验证码的时候需连同输入框
  3. 密码：最少6位，最高不超过30位，允许数字和字母，区别字母大小写。（报错信息：密码不得少于6位）
  4. 旅行社名称：最少6个中文字，最高30个中文字。
  5. 经营许可证号：最高30个字（不限中英文、数字）。
  6. 联系人：最高5个中文汉字。
  7. 座机号码：允许数字、-符号，最高20位。
  8. 传真号码：同上
  9. 电子邮箱：必须包含@符号，最高30位。
  10. 发票地址：省、市、区/县使用现成数据库，仅供用户下拉选择，详细地址上限30个中文字。
  11. 相关资质：请上传经营许可证、法人身份证正反面等景区资质扫描件，支持jpg、png、gif格式，单张图片不得超过2M。
  12. 注册内容判断方式：光标移出该项输入框马上进行判定，正确的给出绿勾提示，错误的给出红字错误提示。有输入上限的到了上限直接禁止输入。（该条适用于所有身份的注册）


  • 旅行社部门（企业）注册。
  1. 该身份注册所需字段在旅行社注册中都有，同样要求。


  • 旅行社部门（个人）注册。
  1. 与旅行社相同的项目，输入规则亦相同。
  2. 身份证号：18位数字和字母（仅限X，可大些也可小写），不可多也不可少。
  3. 相关资质：请上传身份证正反面等相关资质扫描件，支持jpg、png、gif格式，单张图片不得超过2M。


  • 导游注册。
  1. 与旅行社相同的项目，输入规则亦相同。
  2. 导游证号：最高30个字（不限中英文、数字）。
  3. 相关资质：请上传导游证、身份证正反面等相关资质扫描件，支持jpg、png、gif格式，单张图片不得超过2M。


  • 客栈、商户注册。
  1. 与旅行社相同的项目，输入规则亦相同。


  • 审核反馈及后续操作
  1. 用户在提交审核后，系统已经为其生成账号，可在票之家网站登录。
  2. 如果账号正在审核中，登录账号后给出「审核中」弹窗提示，详见原型的「登录」页下方。
  3. 如果账号审核被拒绝，登录账号后进入「重新申请页面」，页面醒目显示被拒绝原因，用户注册时所填信息全部为其自动填充，重新申请时可修改注册身份类型，手机号码不可修改。
  4. 审核通过的用户，可正常登录票之家平台，系统将根据该用户所获得的权限给出相应的页面和功能。

【模块说明-用户审核】
相关原型页面：用户审核

  • 审核页面按照用户的注册身份类型进行分类呈现（请无视景区公司审核页面）。
  • 待审核列表中呈现用户注册的信息，资质图片新开页面呈现。
  • 通过审核
  1. 点击「通过」按钮后，弹窗「权限分配」，权限分为供应商权限和分销商权限，一个用户既可以有供应商权限也可以有分销商权限，可拥有多个供应商权限，只能拥有一个分销商权限。
  2. 用户的注册身份和权限分配没有必然关联。
  3. 支撑平台的权限只是控制该用户是否有对应模块的入口权限，模块内会有细分权限，在模块内进行设置和控制。
  4. 在产品线模块内（如PMS），平台审核通过并赋予了权限的账号有在PMS内建子账号（无需审核）的功能，可以将自己的部分/全部权限分配给创建的子账号。子账号信息需传回平台。
  5. 如果为用户分配的权限里不包含分销商权限，那么点击确定后即完成整个审核操作，用户在待审核列表中消失。（如果在完成权限分配前关闭了网页，那么再进入系统时，该用户依然在待审核列表，需重新进行审核。）
  6. 如果为用户分配的权限里包含了分销商权限，那么在点击确定后进入渠道分配页面，渠道类型、渠道名均通过接口在「票之家代供销平台」获取。渠道名为单选、必选，点击确定后视为完成整个审核操作。（如果在完成渠道分配前任何一步关闭了网页，那么再进入系统时，该用户依然在待审核列表，需重新进行审核。）
  7. 审核通过后系统自动为注册的手机号码发送短信：「联系人」您好，您在票之家平台注册的「注册身份」账号已经通过审核，可以在www.piaozhijia.com网站登录使用了，感谢您对票之家平台的支持。

  • 拒绝审核
  1. 点击「拒绝」按钮后，弹出拒绝理由录入框，拒绝理由为必填，200个中文字上限。
  2. 拒绝理由完成提交后方视为完成拒绝，在该界面终止/返回操作，用户依然在待审核列表。
  3. 拒绝理由完成提交后，系统自动为注册的手机号码发送短信：「联系人」您好，您在票之家平台注册的「注册身份」账号未能通过审核，原因为「拒绝理由」，请到www.piaozhijia.com使用注册账号登录修改信息后重新提交审核，感谢您对票之家平台的支持。

【模块说明-用户管理】
相关原型页面：用户管理

  • 景区公司因为角色特殊，放到用户管理中为其生成账号。
  • 景区公司必然关联景区，所以该模块还具有景区管理功能。
  • 下一个版本，这个模块也会有所有注册用户的管理功能（包括已拒绝、已通过）。
  • 增加景区公司（生成景区公司账号）
  1. 登录账号：根据景区公司要求填写，可以有英文、数字、中文、符号，上限为20个中文字。
  2. 管辖景区：一个景区公司可以管辖多个景区，一个景区也可以被多个景区公司管辖，只能选择系统中已经建好的景区。
  3. 其它字段在注册的身份中都有相应的，继承输入要求即可。
  4. 景区公司账号生成后需弹出权限分配页面，如分配了分销商权限还需进行渠道分配，与审核开放注册账号交互、逻辑一致。

  • 管理景区公司
  1. 景区公司列表呈现部门注册内容，具体字段参考原型。
  2. 点击「管理」弹出景区所有信息的页面，该页面有「修改密码」「编辑信息」两个功能。
  3. 「修改密码」弹窗，输入两次一致的新密码，确认后密码即被修改。
  4. 编辑信息：除登录账号以外，所有的信息均可编辑，具体参考原型。

  • 增加景区
  1. 景区名称：20个中文字为上限
  2. 所属地区：具体到「区/县」
  3. 景区类型：普通景区、演艺景区两个选项
  4. 景区评级：非A、A、AA、AAA、AAAA、AAAAA六个选项
  5. 以上所有项目均为必填、必选

  • 管理景区
  1. 景区列表呈现景区注册时的所有信息，并且显示产生了关联的景区公司。
  2. 点击「管理」弹出景区所有列表信息，以及在「景区公司系统」中取回的「景区介绍」、「景区图片」内容。
  3. 「管理」页面有编辑功能，点击编辑后，出景区的关联关系以外，所有的信息均可编辑。编辑后的「景区介绍」「景区图片」将同步回「景区公司系统」。

【模块说明-成员权限】
相关原型页面：成员管理


  • 默认有一个超级管理员账号，负责初始的权限管理
  • 权限管理
  1. 权限名称：10中文字上限，不可重名
  2. URL：必须是某一个导航栏的url
  3. 访问限制：只能访问权限内的url（仅限导航部分）

  • 岗位管理
  1. 岗位名称：10中文字上限，不可重名
  2. 权限：可多选

  • 增加成员
  1. 账号：允许英文和数字，20上限
  2. 密码：6-20位，明文录入
  3. 姓名：5中文字上限
  4. 岗位：单选

  • 管理成员
  1. 账号不可修改
  2. 密码栏默认为空，录入密码并保存视为修改新密码
  3. 姓名、岗位可修改

注销登录：点击该按钮后回到登录页面，账号默认填充，密码为空。
